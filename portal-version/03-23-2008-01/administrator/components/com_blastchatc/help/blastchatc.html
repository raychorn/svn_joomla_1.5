BlastChat client 2.3

Welcome to BlastChat client 2.3 admin backend.

BlastChat menus "Configuration", "Users", "Registration" and "Credits" will allow you to administer your client side of BlastChat service.

After successful installation of BlastChat client 2.3 component you should proceed to registration and register your website with BlastChat service.

If you already registered your website, you should create PUBLIC menu item pointing to BlastChat component, which will allow your website visitors to enter chat from frontend.

Examples of creating url link to access blastchat client from your website frontend

Menu item url link - load client with parameters defined in admin backend for BlastChat component:
Use admin backend to create new menu item url link to BlastChat component

Default url link - load client with parameters defined in admin backend for BlastChat component:
http://www.yourwebsite.com/index.php?option=com_blastchatc

UnDetached url link:
(open chat as a standard component in area designated for components, this will overwrite configuration in admin backend concerning detach option)
http://www.yourwebsite.com/index.php?option=com_blastchatc&d=0

Detached url link
(if user has pop-up windows blocked, he will get prompt to load chat Undetached, this will overwrite configuration in admin backend concerning detach option)
http://www.yourwebsite.com/index.php?option=com_blastchatc&d=1

Room id
(Force user to go directly to room with id value of "rid")
http://www.yourwebsite.com/index.php?option=com_blastchatc&rid=123

Need other adjustments? Customize blastchat component files

default.blastchatc.php
bc_getUserGroup - returns group id of current member; if you use other system to assign groups to your members, adjust this function to let blastchat server which group this member belongs to.
bc_getSessionData - returns session data of current visitor.
bc_getLiveSite - returns full URL of your website.
